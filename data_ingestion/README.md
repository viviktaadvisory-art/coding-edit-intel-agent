# Data Ingestion Pipeline

This pipeline automates the ingestion of data into Agent Platform Vector Search 2.0, streamlining the process of building Retrieval Augmented Generation (RAG) applications.

It orchestrates the complete workflow: loading data, chunking it into manageable segments, and importing the processed data into your Vector Search 2.0 Collection. Embeddings are auto-generated by the Collection's configured embedding model.

You can trigger the pipeline for an initial data load or schedule it to run periodically, ensuring your search index remains current.

## Prerequisites

Ensure you have a default GCP project configured:

```bash
gcloud config set project YOUR_PROJECT_ID
```

`terraform` must be installed for the setup step.

## Getting Started

1.  **Set up Datastore:** Provision the Vector Search 2.0 Collection and enable required APIs.

    ```bash
    agents-cli infra datastore
    ```

2.  **Run Data Ingestion:** Process data and ingest it into the collection.

    ```bash
    agents-cli data-ingestion
    ```

Both commands auto-resolve the project from `gcloud` and derive all other parameters (collection ID, region) from conventions. Use `--project` or `--region` to override.

## Local vs Remote Execution

By default, `data-ingestion` runs the pipeline locally using KFP's `SubprocessRunner`. For CI/CD or production environments, use `--remote` to submit to Vertex AI Pipelines:

```bash
agents-cli data-ingestion --remote
```

Remote mode derives the service account, pipeline root, and pipeline name from project conventions. The underlying `submit_pipeline.py` also supports `--cron-schedule` and `--schedule-only` for periodic execution.

## Testing Your RAG Application

Once the data ingestion pipeline completes successfully, you can test your RAG application with Vector Search 2.0.
